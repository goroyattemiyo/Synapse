# Synapse v1.0 リファレンスガイド

本ファイルは Synapse の実行時に参照する詳細ルール集です。
設計の背景と原則は design_v1.0.md を参照してください。

---

## 1. 出力量ガイドライン

| 複雑度 | コードブロック数 | 1ファイルの目安行数 | テスト数 |
|--------|-----------------|-------------------|---------|
| simple | 2（本体+テスト） | 50-100行 | 5-8件 |
| medium | 3（本体+補助+テスト） | 100-200行 | 8-15件 |
| complex | Phase毎に2-3 | 150-300行/Phase | 10-20件/Phase |

コードブロック1個あたりの上限は300行。超える場合はファイルを分割する。

## 2. 媒体別実装ルール

### 2.1 cli（Python CLIコード）
- 完全に動作するPythonコード
- テストはunittest形式を必ず含める
- 外部ライブラリは最小限、標準ライブラリ優先
- コメントは日本語
- ユーザーの実行環境に合わせる
  - Windows: パス区切り\\、エンコーディングutf-8明示、cp932を避ける
  - Mac/Linux: shebang行を含める
- 終了方法（quit/exit/Ctrl+C）を明確にする

### 2.2 web（HTML/CSS/JS）
- simple/mediumは単一HTMLファイルにCSS/JSを埋め込む
- complexはHTML/CSS/JSを分離可
- レスポンシブデザイン（meta viewport必須）
- 日本語フォント考慮（font-family: 'Hiragino Sans', 'Yu Gothic', sans-serif）
- ブラウザで開くだけで動作すること（ビルド不要）
- カラーパレットは3色以内で統一感を持たせる

### 2.3 doc（Markdown文書）
- 見出し構造を厳守（H1→H2→H3、飛ばさない）
- 想定読者を冒頭に明記する
- 1段落は3-5文を目安
- 箇条書きは1階層まで（ネストしすぎない）
- 図表はテキストベースで表現（Mermaid記法可）

### 2.4 writing（テキスト創作）
- 指定がなければ3000-5000字
- 構成: 導入→展開→転換→結末
- キャラクター初出時に簡潔な描写を入れる
- 会話文と地の文のバランス（会話が6割を超えない）
- 段落分けと場面転換に空行を入れる
- 比喩や五感描写を適度に含める
- 文体の統一（です/ます調 or だ/である調を混在させない）

## 3. 媒体別検証基準

### 3.1 cli 検証基準
- コードが構文エラーなく動作するか
- テストが正常系・異常系の両方をカバーしているか
- エラーハンドリング: try-exceptが適切に配置されているか
- 不要な外部依存がないか
- 入力バリデーションがあるか
- 終了方法（quit/exit/Ctrl+C）が明確か

### 3.2 web 検証基準
- HTMLが正しく構造化されているか（DOCTYPE, lang, charset）
- モバイル表示で崩れないか（viewport, flexbox/grid）
- アクセシビリティ: alt属性、十分なコントラスト、フォーカス可能な要素
- リンク切れ・未実装機能がないか
- JavaScriptエラーが発生しないか

### 3.3 doc 検証基準
- 構成が論理的で一貫しているか
- 想定読者にとって適切な粒度か（専門用語に説明があるか）
- 誤字脱字・表記ゆれがないか
- 目次と見出しが対応しているか
- 抜け漏れ: 背景・目的・手順・注意事項が揃っているか

### 3.4 writing 検証基準
- 物語の構成が破綻していないか（伏線の回収、時系列の整合性）
- キャラクターの言動に一貫性があるか
- 文体が統一されているか
- 読者を引き込む冒頭になっているか
- 結末が唐突でないか

## 4. セキュリティチェック詳細

### 4.1 禁止パターン
- eval(), exec(), compile() の使用（ast.literal_evalは許可）
- os.system(), subprocess の無制限使用（サンドボックス内の限定使用は許可）
- __import__() による動的インポート
- pickle.loads() での信頼できないデータ読み込み

### 4.2 機密情報チェック
- APIキー、トークン、パスワードのハードコード
- .envファイルの内容の直接出力
- 個人情報（メールアドレス、電話番号）の埋め込み

### 4.3 通信チェック
- requests, urllib, http.client による外部通信
- WebSocket接続
- 例外: ユーザーが明示的にAPI連携を要求した場合は許可し、理由をコメントに明記する
